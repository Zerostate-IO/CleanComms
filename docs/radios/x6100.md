# Xiegu X6100 Radio Profile

[Source: official] | [Support Tier: 2 (Community-Validated)](README.md#scope)

The Xiegu X6100 is a portable HF/50MHz SDR transceiver with built-in battery and touchscreen interface. It implements the Icom CI-V protocol in IC-7000 compatibility mode.

---

## Protocol Support

### CI-V (IC-7000 Compatibility Mode)

The X6100 communicates using Icom's CI-V protocol, emulating an IC-7000 transceiver. This provides compatibility with existing CAT software designed for Icom radios.

| Attribute | Value |
|-----------|-------|
| **Protocol Type** | Icom CI-V |
| **Compatibility Mode** | IC-7000 |
| **Hamlib Model ID** | 3074 |
| **Default Baud Rate** | 19200 |

#### Serial Configuration

| Parameter | Value |
|-----------|-------|
| Baud Rates | 4800, 9600, 19200, 38400 |
| Data Bits | 8 |
| Stop Bits | 1 |
| Parity | None |
| Flow Control | None |

> **Source:** [official] X6100 Extended Manual v1.0

## CAT Capabilities

### Supported Commands

The X6100 implements a **subset** of IC-7000 CI-V commands. Confirmed support includes:

- Frequency control (read/write VFO)
- Mode control (USB, LSB, CW, etc.)
- VFO switching (A/B)
- Split mode control
- RF power control
- S-meter and SWR reading

### Notable Gaps

The following IC-7000 commands are **not fully supported**:

| Capability | Status | Notes |
|------------|--------|-------|
| IF Filter Bandwidth | Unsupported | No CI-V filter commands |
| AGC Control | Unsupported | AGC settings not accessible via CAT |
| Memory Channels | Unsupported | No memory channel read/write |
| ALC Metering | Unsupported | ALC readings not available |

> **Source:** [hamlib] Hamlib X6100 backend implementation
>
> **Note:** Official documentation does not provide a complete CI-V command reference. The command subset above is derived from Hamlib source code and community testing.

## PTT Control

The X6100 supports multiple PTT methods:

| Method | Supported | Notes |
|--------|-----------|-------|
| CAT PTT | Yes | Software PTT via CI-V command |
| RTS PTT | Yes | Hardware handshake line |
| DTR PTT | No | Not supported |
| Hardware PTT | Yes | Dedicated PTT jack |
| VOX | Yes | Voice-activated TX |

> **Source:** [official] X6100 Extended Manual

**Recommended:** Use RTS PTT for digital mode operation to avoid potential CAT PTT timing issues.

## Audio

The X6100 provides USB audio for digital mode operation:

| Feature | Value |
|---------|-------|
| USB Audio | Yes (sound card mode) |
| Input Method | USB |
| Output Method | USB |
| Sample Rates | 48kHz, 44.1kHz |
| Input Level Control | Yes (software) |
| Output Level Control | Yes (software) |
| ACC Port | Yes |

> **Source:** [official] X6100 Extended Manual

### Firmware Note

Firmware version **1.1.8+** is recommended for stable USB audio during digital mode operation. Earlier versions may experience audio dropouts.

## Known Issues

### X61-0001: CI-V Mode-Switch Response Delay

| Attribute | Value |
|-----------|-------|
| Severity | Minor |
| Source | community-verified |
| Confidence | Medium |

When switching operating modes via CI-V, the X6100 may exhibit response delays of 200-500ms before the mode change is reflected. This can cause timing issues in automated CAT sequences.

**Workaround:** Add 500ms delay after mode-switch commands before sending subsequent CAT queries.

---

### X61-0002: CI-V Address Not Configurable

| Attribute | Value |
|-----------|-------|
| Severity | Minor |
| Source | community-verified |
| Confidence | Low |

The X6100 uses a fixed CI-V address and does not support address configuration. This may cause conflicts when multiple Icom-compatible radios are on the same CI-V bus.

**Workaround:** Use separate serial interfaces for each radio when operating multiple CI-V devices.

---

### X61-0003: Incomplete IC-7000 Command Compatibility

| Attribute | Value |
|-----------|-------|
| Severity | Major |
| Source | hamlib |
| Confidence | Medium |

While X6100 advertises IC-7000 CI-V compatibility, not all IC-7000 commands are implemented. Commands for IF filter control, AGC settings, and some metering functions are missing or behave differently.

**Workaround:** Verify command support against Hamlib backend source or test manually before relying on specific CI-V commands.

---

### X61-0004: Firmware Update Resets CAT Settings

| Attribute | Value |
|-----------|-------|
| Severity | Minor |
| Source | community-verified |
| Confidence | Medium |

After firmware updates, CAT/serial communication settings may reset to defaults.

**Workaround:** Document CAT settings before firmware updates and verify after restart.

---

## Recommendations

1. **Baud Rate:** Use 19200 baud for optimal CI-V responsiveness
2. **PTT Method:** Prefer RTS PTT over CAT PTT for digital modes
3. **Software:** Hamlib rigctld recommended for command abstraction
4. **Firmware:** Keep firmware updated for USB audio stability
