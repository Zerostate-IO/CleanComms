## CAT Protocol Support

The TX-500 supports **dual CAT protocol modes**, requiring separate capability tracking:

### Protocol Profiles

| Profile | Protocol Type | Description |
|---------|---------------|-------------|
| `ts-2000` | Kenwood TS-2000 | Standard Kenwood-compatible mode for broad software compatibility |
| `lab599-extended` | LAB599 Extended | Proprietary extended mode with additional commands and capabilities |

### Recommendation Policy

**Preferred Profile:** `lab599-extended`

The LAB599 extended protocol provides superior control and additional capabilities not available in TS-2000 mode. Use this profile when:

- Running CleanComms with direct Hamlib integration
- Requiring extended metering (SWR, ALC, power output)
- Needing power control and AGC adjustment

**Fallback Profile:** `ts-2000`

Use TS-2000 mode when:

- Third-party software requires Kenwood standard protocol
- Maximum compatibility with legacy CAT applications is needed

## Known Issues

### TX5-0001: Serial Port Enumeration Race

| Attribute | Value |
|-----------|-------|
| **ID** | TX5-0001 |
| **Title** | Serial Port Enumeration Race |
| **Severity** | Minor |
| **Confidence** | Medium |
| **Source Tier** | community-verified |
| **Affected Versions** | All |
| **Impact** | cat-control |

**Description:** On some Linux systems, the TX-500 USB serial port may not enumerate immediately after power-on. A brief delay (1-2 seconds) may be required before opening the serial port.

**Workaround:** Add a startup delay in configuration or use udev rules with appropriate settle time.

---

### TX5-0002: Extended Mode Requires Explicit Activation

| Attribute | Value |
|-----------|-------|
| **ID** | TX5-0002 |
| **Title** | Extended Mode Requires Explicit Activation |
| **Severity** | Minor |
| **Confidence** | High |
| **Source Tier** | official |
| **Affected Versions** | All |
| **Impact** | cat-control, digital-modes |

**Description:** The TX-500 powers up in TS-2000 compatibility mode by default. Extended mode must be explicitly activated via menu setting or CAT command sequence before extended commands are available.

**Workaround:** Ensure extended mode is enabled in radio menu before using extended profile capabilities.
