# Security Policy

CleanComms is committed to the security of its users. This policy outlines how to report security vulnerabilities and what to expect from the ZeroState-IO maintainers.

## Supported Versions

The following versions of CleanComms are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

### Preferred Method: GitHub Security Advisories
The preferred method for reporting security vulnerabilities is through [GitHub Security Advisories](https://github.com/ZeroState-IO/CleanComms/security/advisories/new). This allows for private communication and coordinated disclosure.

### Backup Method: Email
If you are unable to use GitHub Security Advisories, you can report vulnerabilities via email to **security@zerostate.io**.

## Our Process

1. **Acknowledgment**: We will acknowledge receipt of your report within 48 hours.
2. **Investigation**: We will investigate the report and determine its severity.
3. **Fix**: We will work on a fix for the vulnerability.
4. **Disclosure**: We will coordinate the disclosure of the vulnerability with you.

### Embargo Expectations
We request that you do not disclose the vulnerability publicly until a fix has been released and we have agreed on a disclosure date. This "embargo" period allows us to protect our users by ensuring a fix is available before the vulnerability is widely known.

## Volunteer SLA
CleanComms is maintained by volunteers. We aim for a **5 business days first response target** for security reports, but this is not guaranteed. We appreciate your patience and cooperation in keeping CleanComms secure.
