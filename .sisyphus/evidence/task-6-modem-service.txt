# Task 6: Integrate fldigi lifecycle in app service

## Summary
Integrated fldigi client lifecycle management into the application service layer.
Created ModemService wrapper with health monitoring, reconnection, and mode management.

## Files Modified
- internal/app/modem_service.go - ModemService implementation (existing, enhanced)
- internal/app/modem_service_test.go - Comprehensive tests (existing, enhanced)
- internal/app/app.go - Integration with HTTP server

## Functionality Implemented
1. On startup: probe fldigi connection, ensure PSK31 mode
2. Reconnect with exponential backoff (1s → 30s, 2x multiplier)
3. Health status exposed: ModemHealth{OK: bool, Error: string, Mode: string}
4. Mode mismatch logged but not forced (idempotent SetMode handles this)
5. Status changes logged with structured logging
6. HTTP Server uses modem service via adapter for /api/v1/modem/* endpoints
