### API Response Examples

**GET /health**
```json
{"status":"ok","rigctld":"ok","fldigi":"ok"}
```

**GET /api/v1/rig/status**
```json
{"frequency":14070000,"mode":"USB","ptt":false,"connected":true}
```

**POST /api/v1/rig/ptt** (Request: `{"state":"tx"}`)
```json
{"state":"tx"}
```

**POST /api/v1/rig/ptt** (Invalid state)
```json
{"error":"invalid_state","message":"state must be 'tx' or 'rx'"}
```

**GET /api/v1/modem/status**
```json
{"mode":"PSK31","tx":false,"connected":true}
```

### Design Decisions
1. **Interface-based dependency injection**: RigClient and ModemClient interfaces allow parallel development with Tasks 5/6
2. **Stub implementations**: Included StubRigClient and StubModemClient for testing and development
3. **Graceful degradation**: Health endpoint reports "degraded" status for missing/unhealthy dependencies
4. **Stdlib only**: Used net/http with no external frameworks as required
5. **Validation**: PTT endpoint validates state is exactly "tx" or "rx", returns 400 with helpful message
