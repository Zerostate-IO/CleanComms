## Functionality Implemented

### Coordinator Struct
- `NewCoordinator(rig, modem, logger, config) *Coordinator` - Constructor with configurable timeout
- `SetPTT(tx bool) error` - Blocks if degraded, enforces ready-state ordering
- `GetPTT() bool` - Returns current PTT state
- `GetState() CoordinatorState` - Returns state machine state (RX/TX)
- `IsHealthy() bool` - Returns true if both rig and modem are healthy
- `Start(ctx)` / `Stop()` - Lifecycle management

### State Machine
- StateRX (initial) → StateTX (if healthy) → StateRX (manual or timeout)
- Thread-safe via sync.RWMutex

### PTT Ordering
1. **Enter TX**: Modem TX first, then Rig PTT (modem ready before RF applied)
2. **Enter RX**: Rig PTT first, then Modem TX (RF stopped before modem stops)
3. Rollback on failure: If rig PTT fails, modem TX is rolled back

### Background Watchdog
- Checks every 1 second
- Tracks last TX timestamp
- Forces RX after configurable timeout (default 60s)
- Logs safety event with WARN level when forced

### Error Handling
- `ErrDegraded` - TX blocked because rig or modem is unhealthy
- `ErrNotRunning` - SetPTT called before Start()

## Interface Design

The coordinator defines its own interfaces rather than depending on concrete types:

```go
type RigController interface {
    SetPTT(state bool) (bool, error)
    Health() RigHealthStatus
}

type ModemController interface {
    SetTX(tx bool) error
    Health() ModemHealthStatus
}
```

This allows the coordinator to work with any implementation that satisfies these interfaces, including mocks for testing.
