## PTT Flow

PTT requests now route through the coordinator:
1. POST `/api/v1/rig/ptt` → `handlePTT()`
2. `handlePTT()` → `coordinator.SetPTT(tx)`
3. `coordinator.SetPTT()` checks health:
   - If degraded → returns `ErrDegraded` → HTTP 503
   - If healthy → coordinates PTT with proper ordering:
     - Enter TX: modem first, then rig
     - Exit TX: rig first, then modem

## QA Scenarios

### Scenario 1: Coordinator guards TX when unhealthy
When modem is unreachable:
- `coordinator.IsHealthy()` returns false
- POST `/api/v1/rig/ptt {"state":"tx"}` returns 503 with error:
  ```json
  {
    "error": "ptt_blocked",
    "message": "system degraded: cannot enter TX mode"
  }
  ```

### Scenario 2: Feature flags visibility
Feature flags are visible in health response under `features` key.
V2 signal-id endpoint is disabled by default (`signal_id_v2_enabled: false`).
