## Tests Implemented

### TestHealthContract
- Verifies health endpoint returns JSON with Content-Type
- Checks required fields: status, rigctld, fldigi
- Validates field types are strings

### TestRigStatusContract  
- Verifies rig status endpoint returns JSON with Content-Type
- Checks required fields: frequency (numeric), mode (string), ptt (bool), connected (bool)

### TestModemStatusContract
- Verifies modem status endpoint returns JSON with Content-Type
- Checks required fields: mode (string), tx (bool), connected (bool)

### TestPTTContract
- Accepts "tx" state → returns 200 with state="tx"
- Accepts "rx" state → returns 200 with state="rx"
- Rejects invalid states with 400:
  - Tested: TX, RX, transmit, receive, 1, 0, true, false, ""
  - Error code: "invalid_state"
  - Message: "state must be 'tx' or 'rx'"
- Rejects malformed JSON with 400, error code: "invalid_request"
